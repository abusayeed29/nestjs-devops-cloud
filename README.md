# NestJS DevOps E-Commerce

This repository contains a full-stack e-commerce application with:

- a `frontend` app built with Next.js 16, React 19, Redux Toolkit, and Stripe
- an `api` app built with NestJS, Prisma, PostgreSQL, JWT auth, and Stripe

The project is organized as a simple monorepo with separate frontend and backend applications.

## Project Structure

```text
nestjs-devops/
|- api/        # NestJS backend API
|- frontend/   # Next.js storefront and dashboards
|- .github/    # CI/CD workflow files
```

## Frontend

The frontend is a customer-facing and admin-facing web application built with the Next.js App Router.

### Main frontend features

- landing page with product browsing
- product details pages
- login, registration, forgot password, and reset password flows
- cart and checkout flow
- Stripe payment form integration
- user dashboard with orders and password change
- admin dashboard with order management

### Frontend stack

- Next.js 16
- React 19
- TypeScript
- Redux Toolkit
- Redux Persist
- Axios
- SASS modules
- Stripe Elements

### Frontend scripts

Run these inside [`frontend/package.json`](/d:/DevApps/nestjs-devops/frontend/package.json):

```bash
npm install
npm run dev
npm run build
npm run start
npm run lint
```

### Frontend environment variables

Create `frontend/.env.local` and add:

```env
NEXT_PUBLIC_API_URL=http://localhost:3001
NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY=pk_test_your_key_here
```

The frontend reads:

- `NEXT_PUBLIC_API_URL` for API requests
- `NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY` for Stripe Elements

By default, the frontend runs on `http://localhost:3000`.

## Backend

The backend is a NestJS API for authentication, products, cart, orders, payments, and user management.

### Main backend features

- JWT authentication with access and refresh tokens
- forgot password and reset password flow
- mail service for password reset emails
- Prisma ORM with PostgreSQL
- product, cart, order, payment, category, and user modules
- Stripe payment support
- Swagger setup in the NestJS app
- Docker Compose support for API deployment

### Backend modules

Current API modules include:

- `auth`
- `cart`
- `category`
- `orders`
- `payments`
- `products`
- `users`

### Backend stack

- NestJS 11
- Prisma 7
- PostgreSQL
- JWT / Passport
- Nodemailer
- Stripe
- Jest

### Backend scripts

Run these inside [`api/package.json`](/d:/DevApps/nestjs-devops/api/package.json):

```bash
npm install
npm run dev
npm run build
npm run start:prod
npm run lint
npm run test
```

### Backend environment variables

Create `api/.env` and configure values like:

```env
PORT=3001
DATABASE_URL=postgresql://postgres:password@localhost:5432/nestjs_devops
ALLOWED_ORIGINS=http://localhost:3000

JWT_SECRET=your_access_secret
JWT_EXPIRES_IN=900
JWT_REFRESH_SECRET=your_refresh_secret
JWT_REFRESH_EXPIRES_IN=7d

STRIPE_SECRET_KEY=sk_test_your_key_here
NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY=pk_test_your_key_here

EMAIL_USER=your_email@example.com
GOOGLE_CLIENT_ID=your_google_client_id
GOOGLE_CLIENT_SECRET=your_google_client_secret
GOOGLE_REFRESH_TOKEN=your_google_refresh_token
FRONTEND_URL=http://localhost:3000
```

The backend code also uses:

- `PORT`
- `DATABASE_URL`
- `ALLOWED_ORIGINS`
- `JWT_SECRET`
- `JWT_EXPIRES_IN`
- `JWT_REFRESH_SECRET`
- `JWT_REFRESH_EXPIRES_IN`
- `STRIPE_SECRET_KEY`
- `EMAIL_USER`
- `GOOGLE_CLIENT_ID`
- `GOOGLE_CLIENT_SECRET`
- `GOOGLE_REFRESH_TOKEN`
- `FRONTEND_URL`

## Local Development

### 1. Start the backend

```bash
cd api
npm install
npm run dev
```

The API will be available at `http://localhost:3001` unless you set a different `PORT`.

### 2. Start the frontend

In a second terminal:

```bash
cd frontend
npm install
npm run dev
```

The frontend will be available at `http://localhost:3000`.

## Database and Prisma

The backend uses Prisma and includes migrations under [`api/prisma`](/d:/DevApps/nestjs-devops/api/prisma).

Typical workflow:

```bash
cd api
npx prisma generate
npx prisma migrate dev
```

Make sure your PostgreSQL database is running before applying migrations.

## Docker

The repository includes [`api/docker-compose.yml`](/d:/DevApps/nestjs-devops/api/docker-compose.yml) for running the API container with environment variables supplied from your local configuration.

## API/Frontend Relationship

- the frontend sends requests to the backend through Axios
- authentication is handled by JWT access and refresh tokens
- password reset links are generated by the backend and point back to the frontend
- checkout uses Stripe on the frontend and Stripe secret operations on the backend
